## Objection Handling Notes

### Principles

- Lead with tactical empathy.
- Use labels before defending a point.
- Prefer how and what questions over pushing for yes.
- Aim for "that's right" rather than "you're right."
- Treat objections as information, not rejection.

### Useful Moves

- Mirror key phrases Alex uses to get him to expand.
- Label the concern directly.
- Summarize his view until he feels understood.
- Ask calibrated questions that let him explain the constraint.
- Look for hidden stakeholders or hidden constraints before trying to solve.

### Example Labels

- It sounds like you are trying to avoid adding more complexity.
- It seems like security and approvals could be a major concern.
- It sounds like the last thing you want is another tool that creates more work.
- It seems like reliability and ease of adoption matter more than new features.

### Calibrated Questions For Objections

- How would something like this need to work for it to be useful to you?
- What would make this feel easy rather than heavy?
- How would your team evaluate whether something like this is worth using?
- What would have to be true for you to trust this in your workflow?
- How does a tool like this get approved internally at Keysight?
- How does this affect everybody else who would need to be involved?

### Likely Objections To Be Ready For

- We already have Salesforce and Microsoft.
- We do not want another tool.
- Security or IT will make this difficult.
- The data is hard to access or tightly governed.
- This sounds useful, but I do not have time to adopt something new.
- I am not sure this is a big enough problem to solve right now.
- Why not just use Claude Code?

### Why Not Just Use Claude Code

- Claude Code may be useful for one-off tasks.
- The question is whether Alex needs ad hoc help or a reliable workflow system around repeatable account work.
- The distinction is not model quality. It is whether the work becomes consistent, visible, repeatable, and usable inside a real operating environment.

### Questions For The Claude Code Objection

- How are you using Claude Code today, if at all?
- In your view, what would Claude Code handle well here?
- What parts of the workflow would still depend on you manually driving the process?
- If you used Claude Code for this, how would consistency, follow-up, and team visibility work?
- How would that fit into your actual day-to-day with Salesforce, email, and internal coordination?
